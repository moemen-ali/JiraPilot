1. Project Overview

Name: agentdesk-mcp | Version: 0.0.1

A sophisticated NestJS REST API that orchestrates multi-agent AI workflows. The backend bridges Jira ticket management with AI agents (powered by OpenRouter) to automate documentation generation, summaries, and requirements.

Core Responsibilities

  • Provide RESTful endpoints for Jira data retrieval (sprints, tickets, epics)
  • Manage dynamic agent configurations loaded from markdown prompt files
  • Orchestrate end-to-end agent runs: fetch Jira data → build context → stream AI responses
  • Stream AI-generated content via Server-Sent Events (SSE) for real-time client experience
  • Enforce authentication via Jira token and OpenRouter API key validation

Key Features

Feature Implementation Use Case
Jira Integration HttpModule + axios for Jira REST API v3 Fetch sprint issues, specific tickets, epic hierarchies with JQL
AI Agents Markdown-based prompts with YAML frontmatter + gray-matter parser Release notes, sprint summaries, ticket summaries, PRD generation
Streaming SSE protocol; async generator pattern for OpenRouter responses Real-time AI output to frontend without waiting for completion
Validation Global ValidationPipe + class-validator decorators Whitelist, forbid unknown properties, type transformation

7. Guards, Filters & Validation Pipes

JiraTokenGuard (Global)

Enforces token-based authentication on all routes except /health and /agents/list.

  • Validates: X-Jira-Token header (non-empty string)
  • Extra check for /agents/run: Also requires X-OpenRouter-Key
  • Error: UnauthorizedException with helpful message

AllExceptionsFilter (Global)

Catches all unhandled exceptions and returns consistent JSON error shape.

{
  "statusCode": 400,
  "message": "Error details",
  "path": "/jira/sprint",
  "timestamp": "2026-04-18T..."
}

ValidationPipe (Global)

Applied in main.ts; validates all incoming request bodies against DTOs.

  • whitelist: true — Strip unknown properties
  • forbidNonWhitelisted: true — Reject unknown properties
  • transform: true — Auto-transform primitives to types

13. Code Patterns & Notable Observations

1. Dependency Injection

All services are injectable singletons. Controllers receive dependencies via constructor injection. NestJS DI container auto-registers and provides instances.

2. DTO Validation

Class-validator decorators on all DTOs. Global ValidationPipe enforces whitelist + transforms primitives. Result: clean, type-safe request handling.

3. Error Handling Strategy

Error Type Handler Response
Jira 401/403 JiraService UnauthorizedException with hint
Jira other errors JiraService BadGatewayException
OpenRouter errors Streamed to client SSE error frame
Unhandled exceptions AllExceptionsFilter Normalized JSON error

4. Streaming Pattern (/agents/run)

  • Set SSE headers (Content-Type, Cache-Control, etc.)
  • Iterate OpenRouterService async generator
  • Write data: {...}\n\n chunks to response
  • Close with [DONE] sentinel

5. Prompt Management

YAML frontmatter (gray-matter) + markdown content in single file. Metadata (name, description, inputs) separate from system prompt. Clean, extensible design.

6. Jira ADF Parsing

Custom recursive tree walker converts Atlassian Document Format → plain text. Handles nested content arrays, extracts text nodes.

7. Flexible Model Selection

Clients can override model via X-Model header; defaults to qwen/qwen3-coder:free. Enables A/B testing, cost optimization.

14. Security Considerations

Strengths

  • Token-Based Auth: X-Jira-Token and X-OpenRouter-Key required on protected routes
  • CORS Whitelist: Origin validation via ALLOWED_ORIGIN env var
  • Input Validation: Global ValidationPipe with whitelist + forbid unknown properties
  • DTO Constraints: Class-validator min/max bounds, array size limits
  • Consistent Error Responses: No sensitive data in error messages

Potential Improvements

Rate Limiting: No rate limiter on Jira or OpenRouter endpoints. Consider @nestjs/throttler to protect API quotas.
Request Logging: No audit trail for debugging or compliance. Add middleware to log requests (tokens redacted).
JQL Injection: JQL queries built with string interpolation. Form-controlled, but could add parameterized JQL or stricter validation.
Token Rotation: Tokens passed per-request; no refresh mechanism. Client must manage lifecycle.

15. Recommendations & Roadmap

High Priority

Expand E2E Tests: Current test is a placeholder. Add integration tests for all endpoints, error paths, and full agent orchestration flow with mocked Jira & OpenRouter.
Request Logging Middleware: Implement structured logging for all requests (with token redaction). Essential for debugging, audits, monitoring.
JQL Safety Validation: Consider additional guards on JQL queries or parameterized search patterns. Currently form-controlled, but could be tighter.

Medium Priority

Rate Limiting: Add @nestjs/throttler to protect Jira and OpenRouter API quotas. Prevent abuse, ensure fair usage.
Streaming Resilience: Implement client-side retry logic and event IDs for resumable SSE streams. Handle network interruptions gracefully.
Agent Extensibility: Support dynamic agent registration (not just file-based). Version prompts, allow hot-reload.

Low Priority

Metrics & Observability: Add Prometheus metrics (request latency, token usage). Consider OpenTelemetry for distributed tracing.
OpenRouter Model Validation: Validate model strings against known models before sending. Cache model list periodically.
Secrets Management: Move tokens from headers to API keys or OAuth. Consider HashiCorp Vault or cloud-native secrets service.
