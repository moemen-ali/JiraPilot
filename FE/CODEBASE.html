1. Project Overview

AgentDesk is a modern React frontend built with Next.js 14.2 for orchestrating AI agents. It provides a sophisticated three-pane dashboard interface where users can:

  • Browse and select from a curated list of AI agents
  • Fill in agent-specific input forms
  • Execute agents with real-time streaming output
  • Monitor tool execution (MCP calls) in real-time
  • Export agent output as formatted PDFs
  • Manage API credentials through a secure settings panel
Status: Active development. Bootstrap via create-next-app, utilizing Next.js App Router for routing and React hooks for state management.

  • Theming: data-theme="dark" attribute on <html> element; toggled at runtime

Client-Side Architecture

The entire application is a Single-Page Application (SPA) after hydration. Key characteristics:

  • Marked with 'use client' directive in page.jsx
  • All state managed via React hooks (no external store)
  • No server-side rendering or data fetching at page level
  • Credentials stored in browser localStorage
  • Backend calls via HTTP fetch (SSE for streaming)

Key Interactions

  1. Agent Selection: Click agent in middle list → updates activeId → resets inputs/output
  2. Form Submission: Fill inputs → click "Run agent" → calls useAgentRunner.run()
  3. Streaming Output: SSE stream from backend → accumulates output state
  4. PDF Export: On completion, auto-opens modal → user clicks "Download" → triggers exportPdf()
  5. Settings: Click "Settings" nav item → SettingsBody renders form → saves to localStorage
  6. Theme Toggle: Click moon/sun icon → updates DOM + localStorage

10. Notable Patterns & Observations

Architecture Patterns

  • Monolithic Page: All UI logic lives in /app/page.jsx (1,200+ lines). No component extraction despite size.
  • Client-Only SPA: Entire app runs client-side; no SSR or server-side data fetching.
  • Streaming Execution: Backend integration uses Server-Sent Events (SSE) for real-time agent output.
  • Dependency-Free Components: Custom Markdown renderer and icon SVGs reduce external dependencies.

State Management

  • React hooks only (useState, useRef, useCallback, useEffect)
  • No Redux, Zustand, or context API
  • State is local to Home component; hooks encapsulate execution logic

Styling Approach

  • All inline React style={} objects with CSS custom properties
  • No CSS Modules, no build-time CSS-in-JS (Styled Components, Emotion)
  • No Tailwind CSS (despite geist dependency)
  • Theme toggle via runtime DOM attribute change + localStorage

API Integration

  • Single backend endpoint: POST /agents/run (SSE response)
  • No REST CRUD operations
  • Credentials passed as HTTP headers (X-Jira-Token, X-OpenRouter-Key, X-Model)

Form Handling

  • Manual state (inputs object) instead of react-hook-form
  • No validation framework
  • Reset via seed object from agent metadata

Unused/Dead Code

  • geist dependency: installed but not used (custom tokens override)
  • react-markdown dependency: installed but custom renderer used
  • Three components listed in file structure don't exist (InputForm, OutputPanel, AgentSelector)
  • RECENT_RUNS mock data: hardcoded, never updated
  • /settings page: scaffold, not fully integrated

Performance

  • Layout: CSS Grid (3-pane) is efficient
  • List Rendering: Agent list is small (4 items), no virtualization needed
  • Bundle Size: Custom components + minimal dependencies help keep bundle lean
  • Markdown: Inline regex-based parser is lightweight but has limited feature set

11. Recommendations

High Priority

  1. Refactor page.jsx: Extract components into separate files
    • components/NavAside.jsx
    • components/RunPanel.jsx
    • components/PdfPreviewModal.jsx
    • components/SettingsBody.jsx

    This will improve maintainability and enable code reuse.

  2. Add test suite: Unit tests for hooks, integration tests for agent execution
    • Jest + React Testing Library setup
    • Focus on useAgentRunner, useAgentList, mcpClient
    • E2E tests with Playwright or Cypress for full workflows
  3. Implement settings page: /settings is currently a scaffold
    • Fully integrate with root Home state or create standalone context
    • Add form validation for API keys
    • Show connection status indicators
  4. Secure credentials: Move from localStorage to backend
    • Implement OAuth or session-based auth
    • Encrypt sensitive keys server-side
    • Remove plaintext secrets from browser

Medium Priority

  1. Clean up dependencies: Remove unused packages (geist, react-markdown) or document rationale
  2. Enhance Markdown parser: Support blockquotes, code blocks with syntax highlighting, links, images
  3. Error boundaries: Wrap components in React Error Boundaries for graceful error handling
  4. Accessibility: Add ARIA labels, keyboard navigation, focus management
  5. Loading states: Improve UX during agent load, form submission

Low Priority

  1. PWA support: Add manifest.json and service worker for offline capability
  2. Internationalization: Integrate i18n if expanding to multiple languages
  3. Analytics: Add event tracking (agent runs, errors, theme toggles) for product insights
  4. Mobile responsiveness: Optimize layout for tablet/mobile screens
